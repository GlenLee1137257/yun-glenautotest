# 📊 Glen自动化测试平台 - 项目分析介绍

**更新时间**: 2026-01-18

**项目类型**: 企业级自动化测试平台

---

## 📋 项目概述

Glen自动化测试平台是一个基于Spring Cloud微服务架构的企业级自动化测试平台，支持接口自动化测试、UI自动化测试、压力测试等多种测试场景。

## 🎯 项目亮点

1. **微服务架构**: 基于Spring Cloud微服务架构，服务解耦，易于扩展
2. **容器化部署**: Docker容器化部署，环境一致性高
3. **权限管理**: 基于Sa-Token的细粒度权限控制
4. **多场景支持**: 支持接口、UI、压力测试多种场景
5. **异步任务**: 基于Kafka的异步任务处理
6. **文件存储**: MinIO对象存储，支持大文件上传

---

## 📈 技术难点

1. **跨服务数据查询**: 项目列表需要查询账号服务获取管理员名字
2. **权限控制**: 网关层统一权限校验，基于角色的权限控制
3. **文件上传**: MinIO集成，大文件分片上传
4. **异步任务**: Kafka消息队列，任务异步执行
5. **服务注册**: Nacos服务注册与发现，配置中心

---

## 🔄 项目重构

### 重构内容

1. **服务拆分**: 从单体应用拆分为微服务
2. **权限系统**: 集成Sa-Token权限框架
3. **前端重构**: Vue2升级到Vue3
4. **容器化**: Docker容器化部署
5. **配置管理**: Nacos配置中心

### 重构收益

- 服务解耦，易于扩展
- 部署灵活，支持容器化
- 权限管理更精细
- 前端性能提升
