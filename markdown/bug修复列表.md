### 子服务器Engine服务无法启动
原因：主服务器的安全组未开Redis的端口入向规则，
修复：增加规则即可

### 用户输入 test 时，登录失败
原因：前端会设置 identityType='phone'，但数据库是 'mail'。
修复：调整登录逻辑，支持多类型查询：如果identityType为空，默认使用mail

### 公共菜单-项目管理-编辑取消还原原始数据
原因：取消编辑时未保存原始数据快照，导致无法还原
修复：在编辑时保存原始数据深拷贝，取消时还原

### 接口管理-接口模块前端调用接口
原因：前端调用：/api/v1/api_module/del
修复：后端 Controller 的删除路由从 /delete 改为 /del

### 测试报告搜索框问题
原因：前端传递字符串日期时间，后端直接与时间戳字段比较导致类型不匹配
修复：将字符串日期转换为 Date 对象，使用 gmt_create 字段进行时间范围查询

### 测试报告批量删除失败
原因：report_detail_api 表不存在，且 ReportDetail*DO 模型类未指定数据库前缀
修复：创建 report_detail_api 表，修复所有 ReportDetail*DO 的 @TableName 注解添加数据库前缀

### 接口自动化断言中变量未替换导致断言失败
原因：断言时预期值中的变量（如 {{category_id}}）未进行替换，直接与实际值比较
修复：在 ApiAssertionUtil.dispatcher 中对预期值调用 ApiRelationGetUtil.getParameter 进行变量替换

### 接口自动化-用例管理-断言配置与JSONPath表达式易混淆
原因：文档中断言表达式使用 $.length 而非 $.length()，且前端缺少对关联变量/断言配置的说明提示，容易导致理解偏差和断言失败
修复：将文档中的断言表达式更正为 $.length()，并在前端用例编辑页为“关联变量”增加说明提示、为断言增加“说明”字段，建议使用「验证第一个课程分类ID与步骤1一致」「验证返回课程列表数量大于0」等中文说明

### 接口自动化-课程完整业务流程测试-步骤3路径变量未替换导致请求异常
原因：步骤3接口地址使用 /courses/{{course_id}}，后端只对查询参数、请求体做变量替换，未对路径做 {{变量名}} 替换，导致 RestAssured 发送请求时路径中包含非法字符“{”“}”，抛出 URISyntaxException，响应为空
修复：在 ApiWireUtil.wireBase 中对 path 调用 ApiRelationGetUtil.getParameter 先做变量替换，再与 base 组合，从而支持在路径中使用 {{course_id}} 这类关联变量

### 接口自动化-用例管理-步骤序号显示从 0 开始不直观
原因：用例编辑页中步骤列表使用 Table 的 index 直接作为显示值，导致“步骤0/步骤1/...”与文档中的“步骤1/步骤2/...”不一致，易混淆
修复：在 `NewOrEditSteps.vue` 中仅在展示层对 num 列使用 `index + 1` 显示，从 1 开始计数，保持与实际执行顺序和文档说明一致，底层存储仍按 0 开始，不影响后端逻辑

### 测试报告-接口测试-详情页步骤序号与用例管理不一致
原因：接口测试报告详情页中步骤序号直接显示数据库中的 num 字段（从 0 开始），且列表表格中未单独处理 num 列，导致报告中的“步骤0”与用例编辑页、“文档描述”不一致
修复：在 `report/interface/details.vue` 中统一处理序号展示逻辑：列表表格中 num 列显示为 `index + 1`，展开详情中的“步骤概览-步骤序号”显示为 `(record.num ?? 0) + 1`，与用例管理和文档保持一致

### 测试报告-接口测试-执行详情页断言结果与实际不一致
原因：前端在报告详情页重新实现了一套断言计算逻辑（JSONPath 解析、正则、变量替换等），与后端断言引擎存在差异，导致部分断言前端显示失败或通过与后端实际结果不一致
修复：重构 `AssertionComparisonTable.vue`，去掉前端对断言的再次计算，仅展示断言配置（来源、类型、动作、表达式、预期值、说明），断言整体结果完全以后端返回的 `assertionState`、`exceptionMsg` 为准，失败时显示后端的详细错误信息，保证展示结果与实际执行一致

### 测试报告-接口测试-断言配置说明列问号图标交互不符合预期
原因：断言配置表格“说明”列使用了问号图标 + 悬浮提示的交互方式，实际业务中说明内容较短且用户希望直接看到完整文字，问号悬浮反而增加了操作步骤
修复：在 `AssertionComparisonTable.vue` 中移除说明列的 `QuestionCircleOutlined` 图标及 Tooltip，仅保留纯文本说明，提升可读性

### 测试报告-接口测试-详情页变量传递与断言展示过于复杂
原因：变量传递链与断言检查在前端实现了较复杂的 JSONPath 解析和变量替换逻辑，既增加维护成本，又容易因实现细节与后端不一致导致误导
修复：对报告详情页 UI 进行重构：
- 新增 `VariableChainView.vue` 用于图形化展示“本步骤接收的变量”和“本步骤提取的变量”，但不再参与真实断言计算
- 新增 `RequestResponseDetail.vue` 用于结构化展示请求/响应数据（方法、路径、头部、查询、Body）
- 简化 `AssertionComparisonTable.vue`，仅展示配置 + 后端执行结果说明，避免前端重复计算

### MOCK 手册-步骤5 更新课程信息断言表达式导致 JSONPath 解析异常
原因：文档中的断言表达式 `$.price` 在实际配置时存在不可见字符（如复制过程中的全角字符或空格），后端 JsonPath 解析报错 `Could not parse token starting at position 2`
修复：在排查后确认响应数据字段 `price` 正常，指导在用例编辑页中手动删除并重新键盘输入标准表达式 `$.price`，并在文档中补充说明：JSONPath 表达式建议手动输入，避免从文档/网页复制导致的不可见字符问题
