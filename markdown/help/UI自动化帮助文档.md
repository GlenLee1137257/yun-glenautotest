## 一、详细操作指南

### 1.1 创建测试用例

**操作路径：** UI自动化 → 用例管理 → 新增UI用例

**必填字段：**

| 字段 | 说明 | 示例 |
|------|------|------|
| **用例名称** | 测试用例的名称 | "用户登录功能测试" |
| **所属模块** | 用例所属的业务模块 | "用户管理" |
| **浏览器** | 执行测试的浏览器 | Chrome（推荐） |
| **执行等级** | 用例的重要程度 | P0（最高优先级） |

**可选字段：**

- **描述**：用例的详细说明
- **标签**：用于分类和搜索

**注意事项：**

- 用例名称要清晰明确，便于后续查找和管理
- 执行等级建议：P0（核心功能）、P1（重要功能）、P2（一般功能）

### 1.2 配置操作步骤

**操作步骤：**

1. **点击"新增操作"按钮**
   - 在用例编辑页面，找到"操作步骤列表"区域
   - 点击 **新增操作** 按钮

2. **填写步骤基本信息**
   - **名称**：步骤的描述性名称（必填）
   - **步骤类型**：选择"本地步骤"（默认）

3. **选择操作类型**
   - 使用**级联选择器**选择操作类型
- 第一级：选择操作分类（浏览器、鼠标、键盘、等待、断言等）
- 第二级：选择具体操作（打开窗口、鼠标左键点击、键盘输入等）

4. **配置操作参数**
   
   根据操作类型，系统会自动显示需要的字段：

   **不需要定位的操作（只需填写"值"）：**
   - 打开窗口：填写 URL 地址
   - 强制等待：填写等待时间（毫秒）
   - 网页标题断言：填写期望的标题文本

   **需要定位的操作（需要填写定位方式、定位表达式、值）：**
   - 键盘输入：定位方式 + 定位表达式 + 输入的值
   - 鼠标点击：定位方式 + 定位表达式
   - 元素断言：定位方式 + 定位表达式 + 期望值

5. **配置可选参数**
   - **描述**：步骤的详细说明（可选）
   - **是否截图**：是否在执行该步骤时截图（建议在关键步骤开启）
   - **失败是否继续**：如果该步骤失败，是否继续执行后续步骤

6. **保存步骤**
   - 点击 **确定** 按钮保存
   - 步骤会出现在步骤列表中


### 1.4 执行测试用例

**操作步骤：**

1. **进入用例管理页面**
   - 点击左侧菜单：**UI自动化 → 用例管理**

2. **找到要执行的用例**
   - 在用例列表中找到目标用例
   - 可以使用搜索框按名称搜索

3. **执行用例**
   - 点击用例右侧的 **执行** 按钮
   - 系统会弹出确认对话框
   - 点击 **确认** 开始执行

4. **等待执行完成**
   - 执行过程中，可以在页面上看到执行状态
   - 执行完成后，会自动跳转到测试报告页面

**执行状态说明：**

- **执行中**：用例正在执行，请耐心等待
- **执行成功**：所有步骤都执行成功
- **执行失败**：有步骤执行失败，查看报告了解详情

**注意事项：**

- 执行过程中不要关闭浏览器标签页
- 如果执行时间过长（超过5分钟），可能是页面加载慢或网络问题
- 执行失败时，查看报告中的错误信息，定位问题

### 1.5 查看测试报告

**报告入口：**

1. **执行完成后自动跳转**
   - 用例执行完成后，会自动跳转到报告详情页面

2. **从报告列表进入**
   - 点击左侧菜单：**测试报告 → UI测试**
   - 在报告列表中找到目标报告
   - 点击用例名称进入详情

**报告内容说明：**

**1. 执行概览**
- 用例名称、执行时间、总耗时
- 总步骤数、成功步骤数、失败步骤数
- 整体执行状态（✅ 成功 / ❌ 失败）

**2. 步骤详情列表**
- 每个步骤的详细信息：
  - 步骤名称、操作类型
  - 执行状态（✅ 成功 / ❌ 失败）
  - 执行耗时
  - 操作参数（定位方式、定位表达式、值等）
  - 截图（如果配置了）
  - 错误信息（如果失败）

**3. 查看截图**
- 如果步骤配置了截图，点击"查看截图"按钮
- 可以查看执行该步骤时的页面状态
- 对于失败的步骤，截图有助于排查问题

**4. 错误信息分析**
- 如果步骤失败，会显示详细的错误信息
- 常见错误类型：
  - 元素定位失败：找不到元素
  - 操作超时：元素未在指定时间内出现
  - 断言失败：实际值与期望值不匹配

### 1.6 编辑和删除步骤

**编辑步骤：**

1. 在步骤列表中，找到要编辑的步骤
2. 点击步骤右侧的 **编辑** 按钮
3. 在弹出的对话框中修改配置
4. 点击 **确定** 保存修改

**删除步骤：**

1. 在步骤列表中，找到要删除的步骤
2. 点击步骤右侧的 **删除** 按钮
3. 在确认对话框中点击 **确认**
4. 步骤会被删除（注意：此操作不可恢复）

**调整步骤顺序：**

- 步骤会按照"排序"字段自动排序
- 执行时按照排序从小到大的顺序执行
- 如果需要调整顺序，可以编辑步骤的"排序"字段

**注意事项：**

- 删除步骤后，后续步骤的排序会自动调整
- 建议在删除前先查看用例，确认不会影响其他步骤
- 编辑步骤时，注意检查定位表达式是否正确

## 二、支持的操作类型

### 2.1 鼠标操作

| 操作 | 说明 | 需要配置的字段 | 使用场景 |
|------|------|--------------|---------|
| **鼠标左键点击** | 单击元素 | 定位方式 + 定位表达式 | 点击按钮、链接 |
| **鼠标右键点击** | 右键点击元素 | 定位方式 + 定位表达式 | 打开右键菜单 |
| **鼠标左键双击** | 双击元素 | 定位方式 + 定位表达式 | 双击文件、列表项 |
| **鼠标移入元素** | 鼠标悬停在元素上 | 定位方式 + 定位表达式 | 触发悬停效果、显示提示 |
| **鼠标拖拽元素到目标元素** | 拖拽元素到目标位置 | 源元素定位 + 目标元素定位 | 拖拽排序、文件上传 |

**使用场景：**
- **左键点击**：最常用的操作，用于点击按钮、链接、复选框等
- **移入元素**：测试悬停效果，如显示下拉菜单、提示信息
- **拖拽**：测试拖拽功能，如拖拽排序、拖拽上传

### 2.2 键盘操作

| 操作 | 说明 | 需要配置的字段 | 使用场景 |
|------|------|--------------|---------|
| **键盘输入** | 在输入框中输入文本 | 定位方式 + 定位表达式 + 值 | 输入用户名、密码、搜索关键词 |
| **清空输入框** | 清空输入框内容 | 定位方式 + 定位表达式 | 清空表单字段 |
| **表单提交** | 提交表单 | 定位方式 + 定位表达式 | 提交表单（通常用点击提交按钮代替） |

**使用场景：**
- **键盘输入**：最常用的操作，用于在输入框中输入文本
- **清空输入框**：在输入新值之前，先清空旧值

### 2.3 等待操作

| 操作 | 说明 | 需要配置的字段 | 使用场景 |
|------|------|--------------|---------|
| **强制等待** | 固定等待指定时间 | 值（毫秒数） | 等待页面加载、等待动画完成 |
| **隐式等待** | 等待元素出现（全局生效） | 值（毫秒数） | 设置全局等待策略 |

**等待时间设置建议：**

- **页面加载**：通常 2-5 秒
- **网络请求**：根据接口响应时间，通常 3-5 秒
- **动画效果**：根据动画时长，通常 1-2 秒
- **元素出现**：如果使用隐式等待，通常设置 5-10 秒

**强制等待 vs 隐式等待：**

- **强制等待**：固定等待指定时间，无论元素是否出现
- **隐式等待**：如果元素立即出现则不等待，如果未出现则等待最多指定时间
- **建议**：优先使用强制等待，简单可控；隐式等待适合元素出现时间不确定的场景

**隐式等待的逻辑说明：**

- **全局生效**：隐式等待一旦设置，会对后续所有步骤生效，直到被重新设置或结束
- **多次设置会覆盖**：如果在步骤2设置了隐式等待为5秒，然后在步骤5又设置了隐式等待为10秒，那么步骤5之后的所有步骤都会使用10秒的隐式等待（之前的5秒设置被覆盖）
- **设置为0时结束隐式等待**：如果设置隐式等待为0，会禁用隐式等待功能，后续步骤不再使用隐式等待

### 2.4 浏览器操作

| 操作 | 说明 | 需要配置的字段 | 示例 |
|------|------|--------------|------|
| **打开窗口** | 打开指定 URL | 值（URL地址） | `https://www.example.com` |
| **关闭窗口** | 关闭当前窗口 | 无 | - |
| **最大化窗口** | 最大化浏览器窗口 | 无 | - |
| **设置窗口大小** | 设置浏览器窗口大小 | 值（宽x高，用逗号分隔） | `1920,1080` |
| **浏览器前进** | 前进到下一页 | 无 | - |
| **浏览器后退** | 返回上一页 | 无 | - |
| **浏览器刷新** | 刷新当前页面 | 无 | - |
| **通过句柄切换窗口** | 通过窗口句柄切换到指定窗口 | 值（窗口句柄） | `CDwindow-xxx` |
| **通过索引切换窗口** | 通过窗口索引切换到指定窗口 | 值（窗口索引，从0开始） | `0`（第一个窗口）、`1`（第二个窗口） |

**浏览器句柄 vs 浏览器索引：**

- **浏览器句柄（Window Handle）**：
  - 是浏览器为每个窗口/标签页分配的唯一标识符，类似于窗口的"身份证号"
  - 每个窗口都有一个唯一的句柄，格式通常是类似 `CDwindow-xxx` 这样的字符串
  - 即使窗口关闭再打开，句柄也会变化（每次打开都是新的句柄）
  - 使用场景：当你需要切换到已知句柄的特定窗口时使用
  - 获取方式：需要通过代码获取窗口句柄（通常在前一个步骤中获取并保存）

- **浏览器索引（Window Index）**：
  - 是按照窗口打开的顺序，从0开始编号
  - 第一个打开的窗口索引是 `0`，第二个窗口索引是 `1`，以此类推
  - 如果窗口关闭，剩余窗口的索引会重新排列（例如：关闭索引0的窗口后，原来的索引1变成索引0）
  - 使用场景：当你需要切换到第几个窗口时使用（按打开顺序）
  - 使用方式：直接填写数字，`0` 表示第一个窗口，`1` 表示第二个窗口

**实际示例：**

假设你打开了3个窗口：
1. 第一个窗口：句柄是 `CDwindow-ABC123`，索引是 `0`
2. 第二个窗口：句柄是 `CDwindow-DEF456`，索引是 `1`
3. 第三个窗口：句柄是 `CDwindow-GHI789`，索引是 `2`

- 如果想切换到第二个窗口：
  - 使用句柄：填写 `CDwindow-DEF456`
  - 使用索引：填写 `1`

- 如果关闭了第一个窗口，剩余窗口的索引会重新排列：
  - 原来的第二个窗口（句柄 `CDwindow-DEF456`）现在索引变成 `0`
  - 原来的第三个窗口（句柄 `CDwindow-GHI789`）现在索引变成 `1`

**使用建议：**
- 如果窗口句柄是已知的（例如从之前的步骤中获取），使用**句柄切换**更精确
- 如果只需要切换到第几个窗口（按打开顺序），使用**索引切换**更简单


### 2.5 断言操作

**浏览器属性断言：**

| 断言类型 | 说明 | 需要配置的字段 | 示例 |
|---------|------|--------------|------|
| **网页标题相等** | 验证标题完全匹配 | 预期值 | `"登录页面"` |
| **网页标题不相等** | 验证标题不完全匹配 | 预期值 | `"错误页面"` |
| **网页标题包含** | 验证标题包含关键词 | 预期值 | `"登录"` |
| **网页标题不包含** | 验证标题不包含关键词 | 预期值 | `"错误"` |
| **网页URL相等** | 验证URL完全匹配 | 预期值 | `"https://example.com/login"` |
| **网页URL不相等** | 验证URL不完全匹配 | 预期值 | `"https://example.com/error"` |
| **网页URL包含** | 验证URL包含关键词 | 预期值 | `"/login"` |
| **网页URL不包含** | 验证URL不包含关键词 | 预期值 | `"/error"` |

**如何查看网页标题：**

- **方法一：浏览器标签页** - 打开网页后，标题会显示在浏览器标签页上，这是最直观的查看方式。
- **方法二：查看页面源代码** - 右键点击页面，选择"查看网页源代码"（或按 `Ctrl+U`），在源代码的 `<head>` 部分找到 `<title>` 标签，标签内的内容就是网页标题。

**元素文本断言：**

| 断言类型 | 说明 | 需要配置的字段 | 示例 |
|---------|------|--------------|------|
| **元素文本大于** | 验证元素文本（数字）大于期望值 | 定位方式 + 定位表达式 + 预期值 | `"100"` |
| **元素文本小于** | 验证元素文本（数字）小于期望值 | 定位方式 + 定位表达式 + 预期值 | `"50"` |
| **元素文本相等** | 验证元素文本完全匹配 | 定位方式 + 定位表达式 + 预期值 | `"登录成功"` |
| **元素文本不相等** | 验证元素文本不完全匹配 | 定位方式 + 定位表达式 + 预期值 | `"登录失败"` |
| **元素文本包含** | 验证元素文本包含关键词 | 定位方式 + 定位表达式 + 预期值 | `"成功"` |
| **元素文本不包含** | 验证元素文本不包含关键词 | 定位方式 + 定位表达式 + 预期值 | `"失败"` |

**什么是"元素"和"元素文本"：**

- **元素**：指网页中的 HTML 元素，如按钮、文本、标签、链接等。例如：`<button>登录</button>`、`<div>欢迎使用</div>`、`<span>用户名</span>` 等。
- **元素文本**：指元素在页面上显示的文字内容。例如：按钮上显示的文字、提示信息、页面上的文本内容等。

**如何查看元素文本：**

- **方法一：浏览器开发者工具** - 按 `F12` 打开开发者工具，点击左上角的"选择元素"图标（或按 `Ctrl+Shift+C`），然后点击页面上的元素，在 Elements 面板中可以看到该元素的 HTML 代码，标签之间的内容就是元素文本。例如：`<button id="login-btn">登录</button>` 中的 `登录` 就是元素文本。
- **方法二：直接查看页面** - 元素文本就是页面上直接显示的文字内容。例如：按钮上显示的文字、提示信息、页面上的文本等，这些就是元素文本。

**如何配置定位表达式：**

元素文本断言需要配置**定位方式**和**定位表达式**来找到要验证的元素。定位表达式的写法与操作步骤中的定位表达式相同，具体写法请参考 [2.6 支持的定位类型](#26-支持的定位类型)。

**配置示例：**

假设要验证一个提示信息的文本，HTML 代码为：`<div id="message">登录成功</div>`

| 定位方式 | 定位表达式 | 说明 |
|---------|-----------|------|
| ID | `message` | 使用元素的 id 属性 |
| CSS_SELECTOR | `#message` | 使用 CSS 选择器 |
| XPATH | `//div[@id='message']` | 使用 XPath 表达式 |
| TAG_NAME | `div` | 使用标签名（不推荐，可能定位到其他 div） |

**完整配置示例：**
- 定位方式：`ID`
- 定位表达式：`message`
- 断言类型：`元素文本相等`
- 预期值：`登录成功`

**元素状态断言：**

| 断言类型 | 说明 | 需要配置的字段 | 使用场景 |
|---------|------|--------------|---------|
| **元素存在** | 验证元素存在于页面 | 定位方式 + 定位表达式 | 验证登录后用户菜单出现 |
| **元素不存在** | 验证元素不存在于页面 | 定位方式 + 定位表达式 | 验证删除后元素消失 |
| **元素启用** | 验证元素可交互 | 定位方式 + 定位表达式 | 验证按钮可点击 |
| **元素禁用** | 验证元素不可交互 | 定位方式 + 定位表达式 | 验证按钮禁用状态 |
| **元素可见** | 验证元素可见 | 定位方式 + 定位表达式 | 验证提示信息显示 |
| **元素不可见** | 验证元素不可见 | 定位方式 + 定位表达式 | 验证加载完成后加载动画消失 |
| **元素被选中** | 验证元素被选中（如复选框、单选框） | 定位方式 + 定位表达式 | 验证复选框已勾选 |
| **元素未被选中** | 验证元素未被选中 | 定位方式 + 定位表达式 | 验证复选框未勾选 |

**什么是"元素状态"：**

- **元素状态**：指元素在页面上的各种状态属性，包括：
  - **存在/不存在**：元素是否存在于页面 DOM 中
  - **启用/禁用**：元素是否可以交互（如按钮是否可以点击）
  - **可见/不可见**：元素是否在页面上可见（可能存在于 DOM 但被隐藏）
  - **选中/未选中**：复选框、单选框等元素是否被选中

**如何查看元素状态：**

- **方法一：浏览器开发者工具** - 按 `F12` 打开开发者工具，使用"选择元素"工具（`Ctrl+Shift+C`）点击页面元素，在 Elements 面板中查看：
  - **禁用状态**：查看元素是否有 `disabled` 属性，如 `<button disabled>提交</button>`
  - **可见性**：在 Styles 面板中查看 `display`、`visibility`、`opacity` 等 CSS 属性
  - **选中状态**：查看复选框/单选框是否有 `checked` 属性，如 `<input type="checkbox" checked>`
- **方法二：直接观察页面** - 通过肉眼观察：
  - **启用/禁用**：禁用的按钮通常显示为灰色，无法点击
  - **可见/不可见**：不可见的元素在页面上看不到
  - **选中/未选中**：选中的复选框/单选框会显示勾选标记

**如何配置定位表达式：**

元素状态断言需要配置**定位方式**和**定位表达式**来找到要验证的元素。定位表达式的写法与操作步骤中的定位表达式相同，具体写法请参考 [2.6 支持的定位类型](#26-支持的定位类型)。

**配置示例：**

假设要验证一个按钮的启用状态，HTML 代码为：`<button id="submit-btn">提交</button>`

| 定位方式 | 定位表达式 | 说明 |
|---------|-----------|------|
| ID | `submit-btn` | 使用元素的 id 属性（推荐） |
| CSS_SELECTOR | `#submit-btn` | 使用 CSS 选择器 |
| XPATH | `//button[@id='submit-btn']` | 使用 XPath 表达式 |
| CLASS_NAME | `btn-primary` | 如果按钮有 class 属性，可以使用 |

**完整配置示例：**
- 定位方式：`ID`
- 定位表达式：`submit-btn`
- 断言类型：`元素启用`

**断言使用建议：**

- 在关键操作后添加断言，验证操作结果
- 建议在用例的最后一步添加断言，验证整体流程是否成功
- 对于重要的验证点，建议开启截图，方便后续排查
- **相等/不相等**：用于精确匹配验证
- **包含/不包含**：用于模糊匹配验证，更灵活
- **大于/小于**：仅适用于元素文本是数字的场景

### 2.6 支持的定位类型

#### 2.6.1 ID

**说明：** 通过 HTML 元素的 `id` 属性定位元素。具有唯一性、性能最优、稳定性高、表达式简单等特点。

**表达式格式：**
- 格式：直接填写 `id` 属性的值，不需要任何前缀或符号
- 示例：HTML `<button id="login-button">登录</button>` → 定位表达式：`login-button`

**使用建议：** 优先使用，如果元素有 `id` 属性，优先使用 ID 定位。避免使用动态生成的 ID（如 `react-id-123456`）。

#### 2.6.2 NAME

**说明：** 通过 HTML 元素的 `name` 属性定位元素，主要用于表单元素。具有语义明确、表单专用等特点，但可能重复。

**表达式格式：**
- 格式：直接填写 `name` 属性的值
- 示例：HTML `<input name="username" type="text">` → 定位表达式：`username`

**使用建议：** 适用于表单元素（input、select、textarea 等）。如果页面中有多个相同 `name` 的元素，会定位到第一个匹配的元素。

#### 2.6.3 CSS_SELECTOR

**说明：** 使用 CSS 选择器语法定位元素，功能强大且灵活，性能良好。

**表达式格式：**
- 格式：使用 CSS 选择器语法（需要点号、#号等符号）
- 常用语法：
  - ID 选择器：`#username`
  - 类选择器：`.btn-primary`
  - 属性选择器：`[type="submit"]`
  - 组合选择器：`.btn.btn-primary`（同时匹配两个 class）
  - 标签+类组合：`button.btn-primary`
  - 后代选择器：`.container .item`
- 示例：HTML `<button class="btn btn-primary" id="submit-btn">提交</button>` → 定位表达式：`#submit-btn` 或 `.btn-primary` 或 `button.btn-primary`

**使用建议：** CSS_SELECTOR 需要点号开头（如：`.btn-primary`），可以组合多个条件；CLASS_NAME 不需要点号（如：`btn-primary`），只能匹配单个 class。避免使用过于复杂的选择器，页面结构变化时容易失效。

#### 2.6.4 XPATH

**说明：** 使用 XPATH 路径语言定位元素，功能最强大但性能较慢，维护成本高。

**表达式格式：**
- 格式：使用 XPATH 语法，通常以 `//` 或 `/` 开头
- 常用语法：
  - 相对路径：`//button`（从任意位置查找）
  - 属性定位：`//button[@id='submit']`
  - 文本定位：`//button[text()='登录']`
  - 包含文本：`//button[contains(text(), '登录')]`
- 示例：HTML `<button id="add-to-cart-sauce-labs-backpack">Add to cart</button>` → 定位表达式：`//button[@id='add-to-cart-sauce-labs-backpack']`

**使用建议：** 只有在其他定位方式都无法使用时，才考虑使用 XPATH。
避免使用绝对路径（如 `/html/body/div[1]/button`）和索引定位（如 `//div[5]/span[2]`），优先使用属性定位。

#### 2.6.5 CLASS_NAME

**说明：** 通过 HTML 元素的 `class` 属性定位元素。表达式简单，但可能重复，只能匹配一个 class。

**表达式格式：**
- 格式：直接填写 `class` 属性中的任意一个值，不需要点号
- 示例：HTML `<button class="btn btn-primary">提交</button>` → 定位表达式：`btn` 或 `btn-primary`（使用其中一个 class 值）

**使用建议：** 适用于元素有 `class` 属性，且某个 `class` 值在页面中唯一时。如果多个元素有相同的 class，会定位到第一个匹配的元素。如果需要同时匹配多个 class，使用 CSS_SELECTOR。

#### 2.6.6 TAG_NAME

**说明：** 通过 HTML 元素的标签名定位元素（如 `button`、`input`、`div` 等）。表达式最简单，但通常不唯一，定位不精确。

**表达式格式：**
- 格式：直接填写 HTML 标签名
- 示例：HTML `<button>提交</button>` → 定位表达式：`button`

**使用建议：**
- ⚠️ **只会定位第一个匹配的元素**：即使页面中有多个相同标签（如6个 `button`），也只会定位到第一个
- ⚠️ **不会操作所有元素**：使用 TAG_NAME 定位 `button` 时，不会点击所有按钮，只会点击第一个按钮
- 实际示例：商品列表页面有6个"加入购物车"按钮时，❌ 错误用法：TAG_NAME 定位 `button` → 只会定位到第一个按钮；✅ 正确用法：ID 定位 `add-to-cart-sauce-labs-backpack` → 精确定位到指定按钮
- 只有在页面中某种标签只有一个元素时，才使用 TAG_NAME 定位。如果需要定位特定元素，优先使用 ID 定位。

#### 2.6.7 LINK_TEXT

**说明：** 通过链接（`<a>` 标签）的完整文本内容定位链接元素。专门用于链接，表达式简单，但文本必须完全匹配。

**表达式格式：**
- 格式：直接填写链接的完整文本内容，必须完全匹配（区分大小写）
- 示例：HTML `<a href="/login">登录</a>` → 定位表达式：`登录`

**使用建议：** 适用于链接文本稳定不变时。文本必须完全匹配，包括空格、标点符号。如果链接文本会变化（如多语言），不建议使用。

#### 2.6.8 PARTIAL_LINK_TEXT

**说明：** 通过链接（`<a>` 标签）的部分文本内容定位链接元素，是 LINK_TEXT 的灵活版本。专门用于链接，灵活性高、容错性好，但可能匹配多个。

**表达式格式：**
- 格式：填写链接文本的部分内容（关键词），不需要完全匹配
- 示例：HTML `<a href="/cart">查看购物车 (3)</a>` → 定位表达式：`购物车` 或 `查看`（数字会变化，但"购物车"是稳定的）

**使用建议：** LINK_TEXT 完全匹配，适用于文本稳定且简短；PARTIAL_LINK_TEXT 部分匹配，适用于文本较长或包含动态内容。如果多个链接包含相同关键词，会定位到第一个匹配的链接。如果链接文本稳定且简短，使用 LINK_TEXT；如果文本较长或包含动态内容，使用 PARTIAL_LINK_TEXT。


**避免使用的定位方式：**

- ❌ 过长的 XPath：`/html/body/div[1]/div[2]/div[3]/form/input[1]`（页面结构变化容易失效）
- ❌ 索引定位：`//div[5]/span[2]`（页面改动易失效）
- ⚠️ 动态生成的 ID：`react-id-123456`（每次刷新可能变化）
